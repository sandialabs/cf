Phenomena, PIRT

PIRT feature permits to describe quantities of interest and associate phenomena and criteria to a created quantity of interest. PIRT view is a stack of tabs starting with the QoI Home View. After that, multiple PIRT tables can be added for each opened QoI.

PIRT content persists across sessions in *.cf file.

QoI Home View

This view lists all the quantities of interest associated to the current ModSim. It is possible to add a new QoI or to modify, open or delete a QoI by selecting it in the table.

Table QoI

The table has four columns: - Creation Date: QoI creation date - Name: the QoI name - Tagged: is QoI tagged? This value can be computed with tag date field. - Tag Date: QoI tag date - Tag Description: The QoI description

Table Events
  • Double-clicking on a QoI row will open it in edition mode in a new tab in a Phenomena View.

Buttons

Add Quantity of Interest

This button adds a new QoI. It throws a new QoI creation wizard.

The user can enter a name for the new QoI. The name can not be null.

The creation date must be stored on the new QoI.

Back

"Back" button permits to return to cf-plugin Home View.

Delete

This button permits to delete the selected QoIs. Multiple selection in QoI table is allowed.

A deletion confirmation dialog should be opened on click.

If the user confirms deletion, the selected QoIs are deleted with all the associated phenomenon groups, phenomena and criteria.

Edit

"Edit" button opens a Phenomena view in a new tab for each selected QoI. Multiple selection in QoI table is allowed.

Copy

"Copy" button duplicates the QoI. The user will be asked for a new QoI name.

Phenomena View

The Phenomena view displays the informations of a quantity of interest. This view is splitted in two tables, quantity of interest header for the QoI informations and phenomena table that contains adequacy columns to assess criteria credibility.

QoI Header Table

The QoI Header table contains QoI informations. The informations are separated in two types, the fixed fields and the variable ones.

The fixed fields are: - Name - Creation Date - Tagged: is QoI tagged? This field is computed depending on tag date. - Tag Date - Tag Description: The QoI description

The variable fields are added depending on the project configuration file. All the variable fields are text fields.

Phenomena Table

This table presents the different phenomenon groups with their phenomena associated. The phenomena has multiples criteria. The informations are separated in two types, the fixed columns and the variable ones.

Fixed columns are: - ID: group or phenomenon ID to display - Phenomena: the name of the group or phenomenon - Importance: the reference Level column to be compared with other Levels criteria

Variable columns are stored as criteria. The criteria can be of different types: Levels or Text.

Levels criteria must be configurable. Their values are stored in the project configuration file.

Table behavior

Editable columns are: - For phenomenon group: - Phenomena - For phenomenon: - Phenomena - Importance - All the variable columns

The variable columns of Levels type must be colored depending of the Importance column. All columns of Level type has a numerical value. The numerical value should be used to compare the Importance column to the criterion column.

The difference is splitted in three level differencing colors. This colors are configured in the project configuration file.

Importance Numerical Value* - *Level Criterion Numerical Value: - >= 0 - = -1 - <= -2

See Generate Configuration for Credibility Project for more details.

Buttons

Back

Return to home page.

Add Phenomenon Group

Adds a new phenomenon group.

Required fields for phenomenon group are: - ID: this field is autogenerated. It is the alphabetic ID of the group. First group starts with 'A', then 'B',etc... - Label: this is the group label or name

Add Phenomenon

Adds a new phenomenon.

Fixed fields for phenomenon are: - ID: this field is autogenerated. It is the alphabetic ID of the group with the phenomenon number for this group (e.g. A1,B3,C2,etc...). - Label: this is the phenomenon label or name - Importance: this is the importance level to compare with adequacy columns

Delete

This button permits to delete the selected Group or Phenomenon. Multiple selection in PIRT table is not allowed.

A deletion confirmation dialog should be opened on click.

If the user confirms deletion, the selected Group or Phenomenon is deleted with all the associated phenomena and criteria.

Tag

Tags the current Quantity of Interest with all its Phenomenon Group, Phenomena and Criteria.

A hashtag should be generated to identify the tagged qoi and tag date should be saved.

The tagged QoI can not be modified or resetted, but it can be deleted.

Reset

Resets the current Quantity of Interest: - Delete all variable QoI Headers - Delete all Phenomenon groups - Delete all Phenomena - Delete all Criteria

Close

This button closes the current Quantity of Interest tab.
