Assess, PCMM

PCMM feature first goal is to associate evidence to the different element or subelement defined, and to assess this evidence. This assessments can be aggregated to display an aggregated score for the different tested elements of the model. This aggregation can be shown as a table or as a radar chart.

Assessor and Role

Assessor: Every user can manage evidence and assess but it has to define his role before. The host machine user id is saved as the user id for the PCMM feature.

Role: The role is used to determine the implication and the credibility of the assessor.

One user can give only one assessment per role, but he can assess multiple times with different roles.

PCMM Mode

The PCMM feature is available in different modes: - DEFAULT: In this mode, the user can assess and manage evidence at the subelement level. - SIMPLIFIED: In the simplified mode, the user assess and manage evidence at the element level. There is no subelements.

This mode is activated depending of the requirement file specified while creating the new credibility process.

If the configuration file associates levels to the PCMM element, even if the PCMM element has subelements, the simplified mode is activated. Otherwise the default mode is activated.

This mode can not be changed once the credibility process is created.

PCMM Role

To manage evidence and assess, each user needs to define his role in the PCMM. If not defined, the user is prompted to select his role when he tries to access to the evidence or the asses views.

Change PCMM Role

Each user can change his role. If the user wants to, he has to double-click on the Role: section in the top-right corner. The role dialog will be displayed.

PCMM Home View

The PCMM Home view displays the progression of the PCMM and the PCMM wheel with the different elements.

PCMM Progression

The progression is indexed by PCMM element. The progression is divided in two major parts, the evidence and the assessment.

In default mode: The progression of each part is divided by the number of subelements of each PCMM element.

Once an evidence is added or an assessment is made on a subelement, the progression is marked as done.

For example, a PCMM Element has 5 subelements. The progress bar will be divided in 10 parts (5 for the evidence and 5 for the assessment).

In simplified mode: The progression is divided by two. Once an evidence is added or an assessment is made on an element, the progression is marked as done.

PCMM Wheel

The PCMM wheel shows the different PCMM elements in one chart. Clicking on a PCMM Element will popup a contextual menu with two options: - Evidence: to manage evidence for this element. It opens the evidence view. - Assess: to manage assessments for this element. It opens the assessment view.

This sections needs to define a role for the user before being used.

If there is no role defined, the user is prompted to select a role.

PCMM Evidence View

The evidence view allows the user to watch and manage evidence.

The user can see all the evidence but can only manage the ones associated to the PCMM element selected on the PCMM wheel.

The user and role wh created the evidence are referenced in the User and Role columns. It is useful to analyze the credibility of the evidence.

In default mode: The evidence are associated to the subelements.

In simplified mode: The evidence are associated to the elements.

Add Evidence

The user can add an evidence by: - Clicking on the Add button. The user must select the destination subelement or element before. - Dragging and droping a document from the Eclipse explorator to the destination subelement or element. If the element dragged is not valid or if the source is not valid, the interface will show the forbidden icon on the dropped item.

Open Evidence

It is possible to open one or more evidence by: - Double-clicking on the evidence in all columns except the Description one. - Selecting the evidence to open and clicking the Open button. - Selecting the evidence to open and typing "Enter" key on the keyboard.

By default, evidences are opened by the Eclipse product. It is possible to change this behavior by using Eclipse's preferences:

PCMM Aggregate View

The aggregate view aggregates all the assessment by subelement and element. It gives a score level to the element based on the level configured.

Tip: There is as many levels as there are subelements/elements. A numerical code is stored on the level to have a correspondance between the subelement/element level and the aggregation level. This gives a numerical score for the aggregation. This numerical score is then converted to a level score.

In default mode: The aggregation is made for the subelements and the elements.

In simplified mode: The aggregation is only made for the elements.
